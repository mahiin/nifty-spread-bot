📱 Telegram Alerts

The bot sends you Telegram messages for every important event — you don't need to keep the dashboard open. Here's what each alert means:
📊 / 🦋 / ⚡ — New Signal Detected
Sent every scan when a strong signal fires.
Shows: strategy, Z-score, strength, S/L, target, regime.
Action: Review signal, then check the Daily Plan tab. Auto-execute fires automatically if enabled.
✅ ORDER PLACED — New Position Opened
Sent by executor Lambda when a trade is placed.
Shows: trade type, qty/leg, entry curve diff, SL, target.
Action: No action needed. Monitor via Positions tab.
🏁 TARGET_HIT  |  🛑 STOP_LOSS  |  ⚠️ EXPIRY_RISK — Position Closed
Sent when position_monitor closes a trade.
Shows: P&L (approx), reason, symbols.
Action: For STOP_LOSS, review strategy params in Controls tab.
🛑 SPREAD TRADING HALTED — Regime Alert
Sent when VIX > 25, large gap, or event risk detected.
No new positions opened. Existing positions monitored.
Action: Wait for OK/SAFE state before resuming.
✅ Token Refreshed — 8:30 AM daily
Angel One JWT is renewed each morning before market open.
🚨 If you see "Token Refresh FAILED" — scanner won't authenticate. Check Secrets Manager credentials immediately.
🚨 ALARM TRIGGERED — Lambda Error
Sent by CloudWatch when any Lambda has repeated errors.
Shows: alarm name, reason, state change.
Action: Check AWS CloudWatch Logs for the named function.
Setup: To enable alerts, set TELEGRAM_BOT_TOKEN and TELEGRAM_CHAT_ID in deploy/.env before running deploy_lambda.sh.
